# Cursor Assistant Rules

## Verification Protocol
1. Before each response:
   - Review relevant rules sections
   - Check task dependencies and blockers
   - Verify session documentation requirements
2. During task execution:
   - Reference rules for each action
   - Document any deviations with explanation
   - Update dependencies as needed
3. After task completion:
   - Verify all required updates were made
   - Check documentation completeness
   - Confirm dependency updates
4. User can request rule verification at any time by asking:
   - "Verify rules compliance"
   - "Check rule adherence"
   - "Confirm following guidelines"

## Task Management
1. Keep TASKS.md updated as we complete tasks
2. Mark completed tasks with [x] instead of [ ]
3. Add completion date and brief notes for completed tasks
4. Add new tasks as they are identified during development
5. Reorganize tasks if priorities change
6. Review and update dependencies when:
   - A task is completed
   - New tasks are added
   - Task requirements change
   - Blockers are identified or resolved
7. Consider both dependencies and blockers when:
   - Planning next steps
   - Estimating completion times
   - Identifying parallel work opportunities
   - Resolving bottlenecks

## Session Documentation
1. Create a new entry in SESSIONS.md for each development session
2. Include the following in each session entry:
   - Date and time
   - Session goals
   - Completed tasks
   - Created/modified files
   - Important decisions made
   - Next steps
3. Reference previous sessions when context is needed
4. Keep documentation clear and concise
5. Note any dependency or blocker changes in session documentation

## Code Management
1. Follow the established project structure
2. Document all new files and major changes
3. Keep code organized and modular
4. Follow Python and React best practices
5. Maintain consistent naming conventions
6. Consider dependencies when:
   - Creating new components
   - Modifying existing code
   - Adding new features
   - Refactoring code

## Communication
1. Be clear about what changes are being made
2. Explain technical decisions when relevant
3. Ask for clarification when needed
4. Provide options when appropriate
5. Keep the user informed of progress
6. Highlight dependency implications when:
   - Suggesting next steps
   - Reporting progress
   - Identifying potential issues
   - Proposing changes

## Dependency Management
1. Before starting a task:
   - Verify all dependencies are met
   - Check for any blockers
   - Confirm parallel development opportunities
2. When completing a task:
   - Update dependent tasks' status
   - Check if any blockers are resolved
   - Review next steps based on dependencies
3. When adding new tasks:
   - Identify all dependencies
   - Note any potential blockers
   - Update affected existing tasks
4. When encountering issues:
   - Check dependency chain
   - Review blocker status
   - Consider alternative approaches

## Progress Tracking
1. Regularly review task dependencies
2. Update blocker status as needed
3. Track parallel development opportunities
4. Monitor critical path progress
5. Adjust priorities based on dependency changes
6. Document dependency-related decisions

## Rule Adherence Verification
1. I will explicitly state when I'm following a specific rule
2. I will document any rule deviations with explanations
3. I will maintain a log of rule adherence in session documentation
4. I will verify rule compliance before completing any task
5. I will provide transparency about rule-related decisions
6. I will acknowledge and correct any rule violations when pointed out
